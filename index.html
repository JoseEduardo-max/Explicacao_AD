🌳 Árvore de Decisão

Como o modelo pensa e toma decisões

Imagina que você precisa tomar decisões fazendo perguntas. É exatamente assim que a árvore funciona.

🧠 O que ela faz?

  • Divide os dados em grupos
  • Deixa os grupos mais puros
  • Continua até decidir

Separar dados parecidos

Ela vai separando os dados até que cada grupo fique o mais parecido possível.

📦 Impureza

Puro → uma única classe

Impuro → mistura de classes

Objetivo: diminuir a impureza

A árvore sempre tenta transformar um grupo bagunçado em grupos organizados.

🐶 Exemplo

A → Cachorro
B → Cachorro
C → Gato

Grupo impuro

Aqui temos um grupo misturado. E a árvore não gosta disso — ela quer separar.

✂️ Como divide?

  • Idade > 18?
  • Altura < 1.70?
  • Tem pelos?
Cada pergunta é como um corte nos dados. Ela transforma um problema complexo em várias decisões simples.

🎯 Melhor divisão

  1. Testa perguntas
  2. Mede impureza
  3. Escolhe a melhor
A árvore testa várias possibilidades e escolhe a que melhor organiza os dados.

🧮 Gini

Gini = 1 - Σ p²

✔ 0 = perfeito

⚠ 0.5 = bagunçado

O Gini mede o quão bagunçado está um grupo. Se só tiver uma classe, o Gini é zero — perfeito.

🔥 Entropia

Entropy = - Σ p log₂(p)

✔ 0 = organizado

⚠ Alto = bagunça

A entropia mede o nível de desordem dos dados. Quanto mais bagunça, maior a entropia.

⚖️ Ganho de Informação

Quanto a divisão melhora os dados

Escolhe a maior redução de impureza

O modelo escolhe a divisão que mais reduz a bagunça. Ou seja, a pergunta que mais organiza os dados.

🌳 Crescimento

  1. Começa com todos os dados
  2. Escolhe melhor divisão
  3. Divide
  4. Repete
A árvore cresce repetindo esse processo várias vezes.

🧠 Quando para?

  • Nó puro
  • Limite de profundidade
  • Poucos dados
Se não parar, a árvore fica complexa demais.

⚙️ Parâmetros

criterion: gini / entropy

max_depth: profundidade

min_samples_split: mínimo para dividir

min_samples_leaf: mínimo na folha

Esses parâmetros ajudam a controlar o comportamento da árvore.

⚠️ Overfitting

Modelo decora os dados

Erra dados novos

Solução: limitar a árvore

Aqui o modelo aprende até o erro dos dados. Ele para de generalizar.

📉 Underfitting

Modelo simples demais

Não aprende direito

Aqui o modelo nem tenta aprender direito.

📊 KNN vs Decision Tree

KNN → usa distância

Decision Tree → cria regras

Enquanto o KNN compara proximidade, a árvore cria lógica.

🚀 Por que usar?

  • Fácil de entender
  • Rápido
  • Interpretável
É um dos modelos mais fáceis de entender e explicar.

🎤 Resumo

A árvore aprende fazendo perguntas e dividindo os dados até deixá-los organizados.

A árvore aprende fazendo perguntas até encontrar a melhor decisão.
